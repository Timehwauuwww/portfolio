Création de bases de données

Compétence 4

Concevoir et mettre en place une base de données à partir d'un cahier des charges client

Déroulé & explication du projet

Exlpications du jeu :

  1. Critique de la base de données existante + proposition d'un nouveau modèle (incluant les correstions) et sa transformation en modèle relationnel (script SQL)
  2. Répondre aux différentes requêtes (livrable sous forme de script SQL)
  3. Extension de la base de données (diagramme de classe + implémentation SQL) + proposition d'un jeu de données test pour peupler la nouvelle base de données

Contexte

Projet SAÉ S1.04 - Conception d'une base de données football: Conception en binôme d'un système de gestion des championnats de football français. Le projet débute par l'identification des dépendances fonctionnelles et la création d'un diagramme de classes UML représentant clubs, équipes, joueurs, matches et résultats. La seconde partie consiste à traduire ce modèle conceptuel en schéma relationnel avec Tutorial D, en respectant les contraintes d'intégrité référentielles et les règles de nommage établies.

Technologies utilisés

  • Langage UML
  • PostgreSQL
  • Visual paradigm
  • Notion
